# CLAUDE.md

## Project Overview

**E-Balance** is a personal finance management application with two main components:

- **CLI Tool** (root module): Imports bank transactions from Excel and classifies them using ML models
- **Backend API** (`backend/`): Ktor web server providing REST APIs for transaction management and investment portfolio tracking

## Repository Structure

```
e-balance/
├── src/                        # CLI application source
├── classification/             # ML classification module
├── backend/
│   ├── server/                 # Ktor API server
│   ├── core/                   # Shared RPC interfaces
│   └── client/                 # Ktor HTTP client library
├── docker-compose.yml          # PostgreSQL + Backend services
├── settings.gradle.kts         # Root Gradle config
└── gradle/libs.versions.toml   # CLI dependency versions
```

## Tech Stack

### CLI (Root Module)
- **Language**: Kotlin 2.3.0 (JDK 25)
- **CLI**: Clikt 5.0.1
- **Database**: PostgreSQL via Flyway migrations
- **Excel**: Apache POI 5.5.1
- **ML**: DL4J ParagraphVectors or built-in neural network
- **Error handling**: Arrow Core 2.0.1 (`Either` type)

### Backend (`backend/server/`)
- **Language**: Kotlin 2.3.0 (JDK 21)
- **Framework**: Ktor 3.4.0 (Netty)
- **Database**: PostgreSQL + HikariCP + Jetbrains Exposed ORM
- **DI**: Koin 4.1.2
- **Migrations**: Flyway
- **Caching**: Caffeine
- **RPC**: Kotlinx RPC

### Testing
- Kotest 6.1.3 (DescribeSpec style)
- MockK 1.14.9
- Testcontainers 1.20.4 (PostgreSQL)

## Build & Run

### CLI

```bash
# Build and install distribution
./gradlew installDist

# Run commands
./gradlew run --args="import /path/to/transactions.xls"
./gradlew run --args="train --engine neural-network"
./gradlew run --args="export <SPREADSHEET_ID>"

# From distribution (no Gradle)
./build/install/e-balance/bin/e-balance import /path/to/transactions.xls
```

### Backend

```bash
cd backend

# Run with Gradle (JDK 21 required)
./gradlew :server:run

# Build fat JAR
./gradlew :server:buildFatJar

# Run with Docker (starts PostgreSQL + backend)
docker-compose up
```

Server runs on `http://0.0.0.0:8080`.

## Environment Configuration

Create `.env` at the project root (CLI) and `backend/server/.env` (backend). See `backend/server/.env.example` for reference.

Key variables:
```
DATABASE_URL=jdbc:postgresql://localhost:5432/ebalance
DATABASE_USERNAME=ebalance
DATABASE_PASSWORD=ebalance
CLASSIFIER_MODEL_PATH=../nn-model.bin
SERPAPI_API_KEY=<api-key>
```

Configuration priority: `.env` file → System environment → `application.yaml`


### CI Matrix
- **classification-and-cli**: macOS runner, JDK 25
- **backend**: Ubuntu runner, JDK 21

### IMPORTANT — Code quality rule
**Every change to backend Kotlin code MUST compile and pass `./gradlew :server:test` before being considered complete.**

Checklist before finishing any backend change:
1. All files that reference a modified type/signature are updated (domain, application, infrastructure, *and* test files)
2. Test fixtures (helper functions like `anAsset(...)`) include all required constructor parameters
3. MockK stubs match the current method signature — count the parameters in `any(), any(), ...` chains
4. Run `cd backend && ./gradlew :server:test --no-daemon` to confirm zero errors

## Architecture

Clean Architecture pattern throughout:

```
domain/         → entities, repository interfaces
application/    → use cases
infrastructure/ → persistence, web, external services
cli/ or web/    → entry points
```

Error handling uses Arrow's `Either<Error, Success>` with sealed error types.

## Database Schema

PostgreSQL with Flyway migrations under `backend/server/src/main/resources/db/migration/`:

- `V1`: `transactions` table (operated_at, description, value, balance)
- `V2`: `category` table + category_id on transactions (48 predefined categories)
- `V3`: `investment_asset` + `investment_sector_snapshot` tables
- `V4`: `exchange` column on investment_asset

CLI migrations are in `src/main/resources/db/migration/`.

## ML Classifier Engines

Select via `--engine` flag:
- `paragraph-vectors` (default): DL4J ParagraphVectors neural embeddings
- `neural-network`: Built-in Kotlin feedforward neural network (no DL4J runtime)

Model file: `nn-model.bin` (generated by training, not committed to git).
